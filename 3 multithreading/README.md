Необходимо решить классическую задачу о поиске всех возможных вариантов расстановки на доске шахматных фигур. Дана квадратная шахматная доска размером NxN. На доске уже размещено K фигур. Фигуры размещены так, что находятся не под боем друг друга. Необходимо
расставить на доске еще L фигур так, чтобы никакая из
фигур на доске не находилась под боем любой другой
фигуры. Необходимо найти все возможные решения.
Входные данные: файл input.txt. 
На первой строке файла записано единственное число —
количество потоков для решения. 
На второй строке файла записаны три числа: N L K (через
пробел). 
Далее следует K строк, содержащих числа x и y (через
пробел) - координаты уже стоящей на доске фигуры.
Координаты отсчитываются от 0 до N-1.
Выходные данные: файл output.txt. Содержит
единственное число - ответ: количество уникальных
решений задачи. Если не было найдено ни одного
решения, в файл необходимо записать 0.
Выходные данные: файл time.txt. Содержит единственное
число — время работы алгоритма в миллисекундах, без
учета времени считывания входных данных и времени на
создание рабочих потоков.
Для решения задачи программа должна найти все
решения, но не выводить каждое из них, а только
рассчитать количество решений. Алгоритм не должен
вызывать взаимных блокировок и не должен содержать
"активного ожидания".
